v 2.1.0.985 - последняя версия со старым хранением паролей
 (кроме SecurityAdmin - он с момента появления хранит хэш)
 
Начиная с версии 2.1.0.986
 - DbSeeds создает стандартных пользователей и в поле EncodedPassword кладет хэш стандартного пароля 
 
 - RegisterClientAsync ожидает прихода хэша пароля 
   и сравнивает его с таким же хэшем из поля EncodedPassword
   
 - LoginViewModel шлет хэш введенного пользователем пароля  
   
 - UserViewModel при создании нового пользователя шлет хэш пароля
 
 - ChangePasswordViewModel - для проверки старого пароля 
   сравнивает поле EncodedPassword с хэшем введенного
   При отправке нового - шлет его хэшш
   
   
	На МГТС, в Могилеве и Витебске стоят старые версии где хранится в базе хранятся операции
создания пользователей с нешифрованными паролями
	Если был сделан снэпшот, пароль хранится не в виде события а в снимке модели
	
При запуске сервера версий 2.1.0.986 и новее
	сервер переписывает в ини версию,
	на которой был предыдущий запуск в Server -> PreviousStartOnVersion
	и записывает тек версию в General -> Version
	
	далее при инициализации InitializeEventStoreService и Model
	если прямо в этот момент не было сброса БД
		 и предыдущий запуск был на версии старше 2.1.0.986
	то для каждого существующего пользователя создаем команду UpdateUser,
		 где вместо пароля кладем хэш
		 
	в случае перезапуска сервера, Server -> PreviousStartOnVersion будет уже тек версия
